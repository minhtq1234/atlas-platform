## 1. Strategy & positioning
- **Smart Artifact generator first.** Lead with artifact quality, data-source connection, and finish. `Smart = AI-native` (chat to get work done).
- **GreenNode MaaS demand engine.** Atlas is GreenNode's go-to-market: the killer app that makes enterprises *consume tokens* (app ‚Üí data ‚Üí model ‚Üí infra). VNG back-office (HR/Legal/FA) is the reference customer.
- **Sovereignty is the wall.** For target customers, data + inference cannot leave VN ‚Äî so Copilot/ChatGPT are *disqualified*, not out-competed. Atlas wins by being the only sovereign option.
- **Governance is subtle plumbing**, not the pitch. Per-user masking/provenance live *behind* the source picker; the UX never lectures.

## 2. Scope
**In:** artifact generation (5 types), VNG-internal, the web product + BFF + export + agent runtime + GreenNode connection.
**Non-goals (now):** write-back/actions; a forked OpenCode; multi-tenant; native mobile; non-identity-scoped caching.
**Parked (intentional, plugs in later):** the governed **data layer** (FDL MCP / Trino masked views / per-user masking / identity propagation), **SSO**, server persistence/retention, deployment.

## 3. Product / UX

**Reference:** coworker.ai (structure & controls); Atlas keeps its own Strata brand.

**Flow:** Home ‚Üí (compose or pick template) ‚Üí Configure overlay ‚Üí Build (streamed) ‚Üí **Studio** (chat + canvas) ‚Üí Library.

**Screens & components (all built unless noted):**
- **Home** ‚Äî top bar (logo ¬∑ Library ¬∑ **Connect data sources** ¬∑ avatar); hero orb + greeting; **composer**; template gallery; recent artifacts.
- **Composer** ‚Äî free-text brief + control row: **output type** (Doc/Deck/Sheet/Dashboard/Report), **data source** (picker ‚Üí Connect overlay; UI-only for now), **model** (GreenNode picker, 3 models), **üìé attach** (upload; metadata captured, parsing deferred), send.
- **Configure overlay** ‚Äî live per-type preview + chat-to-tailor (applied chips, quick chips incl. *In Vietnamese*) + Build. `role=dialog`, Esc-to-close.
- **Build overlay** ‚Äî real **SSE-streamed** stage labels + eased progress ‚Üí routes to Studio; degraded/offline surfaced as a toast.
- **Studio** ‚Äî chat-left + canvas-right; chat **regenerates into versions** (context-aware via OpenCode session reuse); version history; pager + filmstrip (decks); zoom; View/Edit toggle (edit = via chat); **Download**; Share/Copy (stub).
- **Connect data sources** ‚Äî overlay lists governed sources with connected / connect / **no-access** states + one quiet governance line. **UI-only** (no backend yet).
- **Library** ‚Äî localStorage-persisted recent artifacts; survives reload.
- **States** ‚Äî empty, building, error, permission-denied, oversized-upload, "no governed data."
- **Partial:** bilingual (output language is a build option; UI shell still EN, VN dictionary scaffolded); refreshable reports ("as-of" stamp only); real sharing.

Ports: web `5173` (dev)/`5174` (preview) ¬∑ BFF `8787` ¬∑ artifacts `8742` ¬∑ OpenCode `4096` ¬∑ model stub `8788`.

**Generation seam.** The web depends only on a `GenerationEngine` interface (`setEngine`). A **resilient** engine tries the BFF (incl. SSE streaming) and falls back to an in-browser **mock** if it's unreachable ‚Äî the app always works. The BFF runs `AGENT_RUNTIME=direct` (BFF‚Üímodel) or `opencode` (BFF‚ÜíOpenCode‚Üímodel). All model output is **zod-validated**; on error it falls back to a deterministic **template** and marks the artifact `degraded` (surfaced in the UI).

**Hardening (done):** SSE heartbeat + client behavior; OpenCode **session reuse** (context-aware revise) + abort/timeout + session cleanup; **egress allowlist** (non-loopback must be https + on `MODEL_ALLOWED_HOSTS` ‚Äî sovereignty teeth); input bounds + body limit; prompt-injection delimiters; balanced-brace JSON extraction; truncated upstream errors; xlsx **formula-injection** neutralized; RFC-5987 Vietnamese filenames.

**Components:** `apps/web` (React 18 + Zustand), `services/bff` (Fastify + zod), `services/artifacts` (FastAPI + python-docx/openpyxl/python-pptx), `agent/` (OpenCode `opencode.json` + HR/Legal/FA read-only agents ‚Äî runs headless; **live-verified against GreenNode**: generate + context-aware session-reuse revise through `BFF‚ÜíOpenCode‚ÜíGreenNode`).

## 6. Artifact model
Discriminated union `ArtifactContent` (kind = Doc/Deck/Sheet/Dashboard/Report), mirrored in `apps/web/src/types.ts` and `services/bff/src/types.ts` (zod), and in `services/artifacts/app/models.py` (pydantic). Notables: Doc has labeled bars + callout; **Dashboard `series.bars` are `{label,value}[]`** (per-column categories, e.g. months); Sheet keeps a **live `=SUM`** on export. An `Artifact` wraps versions (`opencodeSessionId`, `degraded` metadata).
> Lesson: when a chat edit "succeeds" but nothing changes, the **schema usually can't represent the request** (not the model) ‚Äî enrich the schema + prompt shape + renderer together.

## 7. GreenNode & models
OpenAI-compatible LLM endpoint `https://maas-llm-aiplatform-hcm.api.vngcloud.vn/v1`; **AIP key** (Bearer) for inference; **IAM service-account** for management. Model = the model's **`path`** (not `code`). Connect/verify: `services/bff/scripts/greennode-connect.sh` + `docs/Connect-GreenNode.md`.
**Enabled chat models (in the picker):** `google/gemma-4-31b-it` (default, ~2s), `qwen/qwen3-5-27b` (VN, reasoning), `minimax/minimax-m2.5`. **Live-verified** in **both `direct` and `opencode`** modes (BFF‚ÜíOpenCode‚ÜíGreenNode). Direct config: `services/bff/.env`; OpenCode provider: `agent/opencode.json` (env-driven baseURL/key). `agent/opencode.json` model ids are the real GreenNode `path`s.

## 11. Decisions
- **Artifact-generation-first**, data layer parked (product pivot). ¬∑ **coworker.ai** = UX reference; keep Strata brand. ¬∑ **Configure, don't fork OpenCode.** ¬∑ **Sovereignty**: no public egress on inference; self-hosted assets; secrets from env/vault. ¬∑ **Read-only** v1. ¬∑ Governance stays subtle in UX.
