## 1. Strategy & positioning
- **Smart Artifact generator first.** Lead with artifact quality, data-source connection, and finish. `Smart = AI-native` (chat to get work done).
- **GreenNode MaaS demand engine.** Atlas is GreenNode's go-to-market: the killer app that makes enterprises *consume tokens* (app → data → model → infra). VNG back-office (HR/Legal/FA) is the reference customer.
- **Sovereignty is the wall.** For target customers, data + inference cannot leave VN — so Copilot/ChatGPT are *disqualified*, not out-competed. Atlas wins by being the only sovereign option.
- **Governance is subtle plumbing**, not the pitch. Per-user masking/provenance live *behind* the source picker; the UX never lectures.

## 2. Scope
**In:** artifact generation (5 types), VNG-internal, the web product + BFF + export + agent runtime + GreenNode connection.
**Non-goals (now):** write-back/actions; a forked OpenCode; multi-tenant; native mobile; non-identity-scoped caching.
**Parked (intentional, plugs in later):** the governed **data layer** (FDL MCP / Trino masked views / per-user masking / identity propagation), **SSO**, server persistence/retention, deployment.

## 3. Product / UX

**Reference:** coworker.ai (structure & controls); Atlas keeps its own Strata brand.

**Flow:** Home → (compose or pick template) → Configure overlay → Build (streamed) → **Studio** (chat + canvas) → Library.

**Screens & components (all built unless noted):**
- **Home** — top bar (logo · Library · **Connect data sources** · avatar); hero orb + greeting; **composer**; template gallery; recent artifacts.
- **Composer** — free-text brief + control row: **output type** (Doc/Deck/Sheet/Dashboard/Report), **data source** (picker → Connect overlay; UI-only for now), **model** (GreenNode picker, 3 models), **📎 attach** (file **parsed to text and fed as generation context** — MD/TXT/CSV/DOCX/XLSX/PPTX/PDF; see §8), send.
- **Configure overlay** — live per-type preview + chat-to-tailor (applied chips, quick chips incl. *In Vietnamese*) + Build. `role=dialog`, Esc-to-close.
- **Build overlay** — real **SSE-streamed** stage labels + eased progress → routes to Studio; degraded/offline surfaced as a toast.
- **Studio** — chat-left + canvas-right, driven by the **agent-skills** layer: every turn returns a typed Action (`clarify | plan | edit | answer`). The chat **edits** (→ new version), **answers** questions, **clarifies** ambiguous asks with tappable **option chips** ("make it longer" → offers concrete options), or proposes a **plan card + Confirm** for big/multi-step changes (adaptive: acts directly when the request is clear). Context-aware via OpenCode session reuse; plan-confirm state persists. Contract/runtime: `services/bff/src/skills/` + spec `docs/superpowers/specs/2026-07-01-agent-skills-design.md`. Version history; pager + filmstrip (decks); zoom; View/Edit toggle (edit = via chat); **Download**; Share/Copy (stub).
- **Connect data sources** — overlay lists governed sources with connected / connect / **no-access** states + one quiet governance line. **UI-only** (no backend yet).
- **Library** — localStorage-persisted recent artifacts; survives reload.
- **States** — empty, building, error, permission-denied, oversized-upload, "no governed data."
- **Partial:** bilingual (output language is a build option; UI shell still EN, VN dictionary scaffolded); refreshable reports ("as-of" stamp only); real sharing.

## 5. Architecture (as-built)

```
apps/web (React/Vite/TS SPA)
  │  Vite dev proxy:  /bff → :8787   /api → :8742
  ▼
services/bff (Node/Fastify/TS)   ── AGENT_RUNTIME ──┐
  /generate  /generate/stream(SSE)  /revise          │
      direct│                            opencode│
            ▼                                     ▼
   GreenNode MaaS (OpenAI-compatible)     agent/ OpenCode server :4096
   …vngcloud.vn/v1  (Bearer AIP key)        │  @opencode-ai/sdk
            └────────► GreenNode MaaS ◄──────┘
                          │  zod-validated ArtifactContent
apps/web renderers ◄──────┘  (5 types)
      │
services/artifacts (Python/FastAPI :8742)  /export → real .docx/.xlsx/.pptx
      (Dashboard/Report → self-contained HTML, client-side)
```
Ports: web `5173` (dev)/`5174` (preview) · BFF `8787` · artifacts `8742` · OpenCode `4096` · model stub `8788`.

**Generation seam.** The web depends only on a `GenerationEngine` interface (`setEngine`). A **resilient** engine tries the BFF (incl. SSE streaming) and falls back to an in-browser **mock** if it's unreachable — the app always works. The BFF runs `AGENT_RUNTIME=direct` (BFF→model) or `opencode` (BFF→OpenCode→model). **Default: `direct`** — simplest, ~2s, no extra server. **`opencode` is the on-demand path** for agents + tools (both are live-verified against GreenNode); its value compounds once the FDL MCP tool is wired, so it's the intended runtime for the governed-data phase. Switch on demand: run `opencode serve` and set `AGENT_RUNTIME=opencode` + `OPENCODE_URL`. All model output is **zod-validated**; on error it falls back to a deterministic **template** and marks the artifact `degraded` (surfaced in the UI).

**Agent mode (built + live-verified, v1).** An opt-in per-conversation toggle routes a build through an autonomous OpenCode tool-loop instead of a single model turn. `POST /agent/plan` proposes a task list (cheap, no sandbox); after the user confirms, `POST /agent/run` (SSE) provisions a **per-session sandbox** (`SANDBOX=local` per-run workdir under the OpenCode project dir in dev; `container` = ephemeral firewalled container in prod), seeds attachment text into `inputs/`, runs the OpenCode **`builder`** agent (`read`+`write` tools), streams normalized `step` events to a **WorkingSteps** panel, and tears the sandbox down. **Capture is file-based:** the builder writes its artifact JSON to `out/artifact.json` with its native `write` tool; the BFF reads it, **zod-validates**, and on failure re-prompts with the errors (bounded self-heal) — no custom OpenCode tool needed. The loop core (`services/bff/src/agent/run.ts`) is **pure/injectable** — always resolves, always destroys the sandbox, and degrades to a template on any failure (session error · no valid artifact · **wall-clock timeout** · step budget · Stop — each with a distinct reason). Deliverable is the **same `ArtifactContent`**, so canvas/versions/export are unchanged. Sovereignty holds: in `container` mode the sandbox's egress is firewalled to the model host; uploaded content stays data-not-instructions. (Note: `bash`/`python` are intentionally off for the builder — our inputs arrive pre-extracted as text, and a headless bash tool blocks on OpenCode's permission prompt; the full-dev-tools tier is a container + stronger-model follow-up.)

**Hardening (done):** SSE heartbeat + client behavior; OpenCode **session reuse** (context-aware revise) + abort/timeout + session cleanup; **egress allowlist** (non-loopback must be https + on `MODEL_ALLOWED_HOSTS` — sovereignty teeth); input bounds + body limit; prompt-injection delimiters; balanced-brace JSON extraction; truncated upstream errors; xlsx **formula-injection** neutralized; RFC-5987 Vietnamese filenames.

**Components:** `apps/web` (React 18 + Zustand), `services/bff` (Fastify + zod), `services/artifacts` (FastAPI + python-docx/openpyxl/python-pptx), `agent/` (OpenCode `opencode.json` + HR/Legal/FA read-only agents — runs headless; **live-verified against GreenNode**: generate + context-aware session-reuse revise through `BFF→OpenCode→GreenNode`).

**Artifact type modules (WS-0, as-built).** Each artifact type is a self-contained module behind Platform registries, so one team can own one type without colliding. BFF: `services/bff/src/artifacts/<type>/{schema,prompt,archetypes,index}.ts` compose via `artifacts/registry.ts` into `ArtifactContent` / `SHAPE` / the archetype registry (`types.ts`/`prompt.ts`/`archetypes.ts` are thin re-export shims — public API unchanged). Web renderers dispatch through `renderers/registry.tsx`; python exports through `exports/registry.py`. **Adding a type = drop one module folder + a couple registry lines.** The `ArtifactTypeModule` contract + decomposition: `docs/superpowers/specs/2026-07-01-artifact-platform-program.md`; authoring guide: `docs/artifact-packs/AUTHORING.md`.

**Exemplar toolkit (phase-2, as-built).** Generation injects a best-effort, capped `<exemplar>` reference block (a few-shot gold doc) fetched from a server-side store — `services/artifacts` `exemplars` table (SQLite, gitignored), ingested from committed seeds + a gitignored `exemplars/` folder via `python -m app.ingest_exemplars`, exposed as `/exemplars`(+`/retrieve`), consumed through BFF `exemplar/provider.ts` and `generateUser`. Framed reference-not-instructions (in `INJECTION_NOTE`). **No exemplar → generation is unchanged.** Safe seeds committed; real docs gitignored (sovereignty).

**Deep generation tier (phase-1).** An opt-in `mode:'deep'` on generate runs a Platform pipeline `Outline → Draft → adversarial rubric-Critic → Revise` (adaptive, cap 2 rounds; `services/bff/src/deep/`) that trades latency for depth. Same `ArtifactContent` out; degrade-safe (outline/draft failure → single-turn; critic/revise failure → keep best); sovereign (all stages via `runModel`). A shared depth rubric anchors the critic and the on-demand eval judge (`RUN_DEEP_EVAL=1`). Fast single-turn stays the default. Phase-2 (parallel decomposition) is future.

## 6. Artifact model
Discriminated union `ArtifactContent` (kind = Doc/Deck/Sheet/Dashboard/Report), mirrored in `apps/web/src/types.ts` and — on the BFF — composed from per-type modules behind `services/bff/src/artifacts/registry.ts` (zod; re-exported by `services/bff/src/types.ts`), and in `services/artifacts/app/models.py` (pydantic). Notables: **Doc** supports an optional **sectioned** structure (`sections[]` = headings + `paragraph`/`bullets`/`numbers`/`table`/`callout`/`bars` blocks) for structured documents, or a flat memo (`paragraphs[]`); it also has labeled bars (vertical, or **horizontal** via `barsLayout`) + callout, and renders inline `**bold**` (→ bold-orange); **Dashboard `series.bars` are `{label,value}[]`** (per-column categories, e.g. months); Sheet keeps a **live `=SUM`** on export. An `Artifact` wraps versions (`opencodeSessionId`, `degraded` metadata).
> Lesson: when a chat edit "succeeds" but nothing changes, the **schema usually can't represent the request** (not the model) — enrich the schema + prompt shape + renderer together. The revise prompt now exposes the exact shape (incl. `barsLayout`) and forbids the agent claiming a change it didn't actually make in `content`.

## 7. GreenNode & models
OpenAI-compatible LLM endpoint `https://maas-llm-aiplatform-hcm.api.vngcloud.vn/v1`; **AIP key** (Bearer) for inference; **IAM service-account** for management. Model = the model's **`path`** (not `code`). Connect/verify: `services/bff/scripts/greennode-connect.sh` + `docs/Connect-GreenNode.md`.
**Enabled chat models (in the picker):** `google/gemma-4-31b-it` (default, ~2s), `qwen/qwen3-5-27b` (VN, reasoning), `minimax/minimax-m2.5`. **Live-verified** in **both `direct` and `opencode`** modes (BFF→OpenCode→GreenNode). Direct config: `services/bff/.env`; OpenCode provider: `agent/opencode.json` (env-driven baseURL/key). `agent/opencode.json` model ids are the real GreenNode `path`s.

## 8. Feature status
**Built:** whole product loop; composer w/ model picker; 5 renderers (dashboards labeled); Configure/Build/Studio; versions + chat-revise; export (docx/xlsx/pptx + HTML); BFF (direct + opencode); **real OpenCode**; **live GreenNode**; resilient fallback; a11y basics + error boundary; egress guard; tests (web ~24 · BFF ~42 · Py ~11); security review + fixes.
**Partial:** bilingual shell; refreshable-report live refresh; Studio inline edit; real sharing; degraded-on-revise surfacing.
**Attachments-as-context (built, Phase 1):** attached files are extracted to text in `services/artifacts` (`/attachments`, SQLite store) and injected into generation via a BFF `ContextProvider` (fills the `context` seam). Verified live on GreenNode (a memo used every fact from an uploaded file). Formats: MD/TXT/CSV/DOCX/XLSX/PPTX/PDF; whole-file inject with size caps. **Phase 2 (RAG, `bge-m3` retrieval for large docs)** is designed + gated by an embeddings spike; the same extractor becomes a `parse_file` tool for the autonomous-agent tier. Spec: `docs/superpowers/specs/2026-07-01-attachments-context-design.md`.

**Multi-step tool agent (built + live-verified, v1):** opt-in "Agent" mode runs an OpenCode-driven plan→read→write→capture loop in a per-session sandbox; plan-gate → autonomous; working-steps UI (task list + tool chips). The BFF loop core, tools, step-normalizer, and endpoints are unit/integration-tested against an **injected mock session + fake sandbox** (BFF 42 · web 24), and the real path is **live-verified against GreenNode** (`SANDBOX=local`): a Deck built from a brief captured `viaModel=true`, `degraded=none` in ~87s, with tool chips streaming. Capture is **file-based** (builder writes `out/artifact.json` → BFF reads + zod-validates + re-prompts on error), which sidestepped the custom-tool synchronous-return gap. Also **verified in the browser** (Home → ✦ Agent toggle → brief → Build → plan-gate → Confirm → live working-steps → Studio); that run surfaced + fixed two integration bugs (`makeResilientEngine` now forwards `agentPlan`/`agentRun`; `/agent/run` aborts on the response-socket close, not `request.raw`, which had killed every run instantly). **Reliability caveat:** gemma-31B is flaky at emitting schema-valid JSON within the self-heal budget — a run may land on the real artifact or gracefully degrade to the template; a stronger model (qwen/minimax) or deeper self-heal is the fix. **Still gated:** the prod `container` sandbox + `atlas-egress` firewall (dev uses `local`); the full `bash`/`python` dev-tools tier (needs a stronger model + OpenCode bash-permission config); live task-list check-off + a wired in-flight Stop. Design/plan: `docs/superpowers/specs/2026-07-01-multi-step-agent-design.md`, `docs/superpowers/plans/2026-07-01-multi-step-agent.md`.

**Structured documents (built + live-verified):** `Doc` evolved into an optional **sectioned** model (headings + `paragraph`/`bullets`/`numbers`/`table`/`callout`/`bars` blocks; flat memos unchanged). A data-driven **archetype registry** ships a `'general'` archetype and drives archetype-aware generation; the brief is **auto-detected** into an editable "Document type" chip in the composer. **Named archetypes are team-owned data, not shipped code** — a team defines its own (e.g. PRD, SOW, policy) by adding a registry entry, no code change. The **sectioned model + generic structured generation are verified on GreenNode**: a structured brief now yields a sectioned document (numbered sections, requirement tables, bullets/numbers) with **no gratuitous chart** (charts are now an explicit `bars` block, not padding). Sectioned `DocView` + docx export (headings/lists/tables). Suites: BFF 51 · web 33 · Python 14. Design/plan: `docs/superpowers/specs/2026-07-01-structured-documents-design.md`, `docs/superpowers/plans/2026-07-01-structured-documents.md`.

**Deferred/parked:** data layer (FDL/Trino/masking/identity); SSO (stub user "Linh"); server storage/retention; multi-tenant; deploy; attachments RAG (Phase 2, gated); attachments on *revise* (v1 = generate only); agent-mode container/live bring-up + self-heal tool-return (gated, above); agent-run **Stop** (× closes the UI; true in-flight fetch-abort is a follow-up).

## 11. Decisions
- **Artifact-generation-first**, data layer parked (product pivot). · **coworker.ai** = UX reference; keep Strata brand. · **Configure, don't fork OpenCode.** · **Sovereignty**: no public egress on inference; self-hosted assets; secrets from env/vault. · **Read-only** v1. · Governance stays subtle in UX.
