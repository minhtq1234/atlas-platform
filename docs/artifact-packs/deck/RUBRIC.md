# Deck Pack — Quality Rubric (Phase 2 · ship gate)

Apply to every output in `EVAL-PROMPTS.md` during Phase 4. **Ship gate: all ★ checks pass
and no fabrication.** Each check is a yes/no a Reviewer can decide from one generated deck.

| # | Check | ★ |
|---|---|:-:|
| 1 | **Assertion titles** — every content-slide title states the takeaway, not a topic. | ★ |
| 2 | **Concision** — ≤ ~40 words body per content slide; no wall-of-text. | ★ |
| 3 | **Narrative arc** — slides follow the archetype's expected flow (see `ARCHETYPES.md`). | ★ |
| 4 | **Grounded** — no fabricated numbers stated as fact; claims attributed or clearly illustrative. | ★ |
| 5 | **Open & close** — cover + TL;DR up front; an explicit ask / next-steps slide at the end. | |
| 6 | **Speaker notes** — substantive per content slide (talking points, not a re-print of the bullets). | |
| 7 | **Right length** — slide count fits the archetype (board ~10–15, pitch ~10–12); not padded, not thin. | |
| 8 | **Tone** — exec-terse vs. persuasive matches the archetype and audience. | |
| 9 | **Variety** — section dividers / varied layouts on longer decks; not N identical bullet slides. | |
| 10 | **Export fidelity** — pptx opens clean: titles as slide titles, bullets as bullets, notes in the speaker-notes pane, dividers distinct. | |

## Scoring
- Score each eval output: ★-count met / total-★, plus the non-★ checks noted.
- **Pass** = all 4 ★ met and check 4 (no fabrication) never violated. Non-★ misses are logged as follow-ups, not blockers — unless a run misses ≥3 non-★, which fails the gate.
- Record the final scorecard (prompt × check grid) in the Phase 4 notes before merge.
