# Atlas — Architecture & Build State (as-built)

**Status:** live pilot build · **Updated:** 1 July 2026 · **Repo:** https://github.com/minhtq1234/atlas-platform
**Supersedes the "future/plan" framing in:** `Plan_Atlas-Web-Phase1.md`, `UX-Spec_Atlas-Home_v1.md`, `Backlog-and-Sprints_Atlas_v1.md` (those remain valid as intent; this doc is what actually exists).

Atlas is a Smart Artifact generator: describe (or pick a template) → a finished, on-brand Doc/Deck/Sheet/Dashboard/Report, refined by chat, exportable as real Office files — generated by **GreenNode MaaS** models.

## 2. Components (as-built)

| Component | Path | Stack | State |
|---|---|---|---|
| Web SPA | `apps/web` | Vite + React 18 + TS + Zustand | **Built** |
| BFF | `services/bff` | Node + Fastify + TS + zod | **Built** |
| Artifact export | `services/artifacts` | Python + FastAPI + python-docx/openpyxl/python-pptx | **Built** |
| Agent runtime config | `agent/` | OpenCode (`opencode.json` + agents) | **Built (config); runs headless, verified vs stub)** |
| GreenNode connection | `services/bff/scripts/greennode-connect.sh` + `.env` | IAM + AIP | **Live-verified** |

### Generation seam
The web depends only on a `GenerationEngine` interface (`setEngine`). A **resilient** engine tries the BFF over HTTP (incl. SSE streaming) and falls back to an in-browser **mock** if the BFF is unreachable — so the app always works. The BFF, in turn, runs `AGENT_RUNTIME=direct` (BFF → model) or `opencode` (BFF → OpenCode → model). All model output is **zod-validated** against the artifact schema; on model error it falls back to deterministic templates and marks the artifact `degraded`.

## 3. Feature status

**Built & working**
- Home: greeting, composer (output-type · **data-source** · **GreenNode model picker (3)** · file attach), template gallery + type tabs, recent artifacts (localStorage).
- Configure overlay (live preview + chat-to-tailor, quick chips incl. *In Vietnamese*).
- Build with **real streamed progress** (SSE) → Studio.
- Studio: chat-left + canvas-right, chat **regenerates into versions** (context-aware via OpenCode session reuse), version history, pager/filmstrip, zoom, View/Edit toggle, **Download**, Share/Copy (stub link).
- 5 artifact renderers (branded HTML); **dashboard charts have per-column labels**.
- **Export**: Doc→.docx, Sheet→.xlsx (live `=SUM`), Deck→.pptx; Dashboard/Report→self-contained HTML.
- **GreenNode live** (direct mode): Gemma 4 31B (default, ~2s), Qwen 3.5 27B (VN/reasoning), MiniMax M2.5.
- Hardening: SSE heartbeat, OpenCode session reuse + abort/timeout + cleanup, **egress allowlist** (sovereignty), input bounds, prompt-injection delimiters, degraded/offline surfacing.
- Tests: web ~22 · BFF ~10 · Python 7. Security review completed, findings fixed.

**Partial**
- **Bilingual**: output language is a build option; the UI shell is still English (VN dictionary scaffolded).
- **Refreshable reports**: "as-of" stamp only — no live re-query flywheel yet.
- **Studio Edit**: edits happen via chat; no direct inline editing.
- **Share**: link is a stub (no real sharing/permissions).
- **OpenCode per-model**: works; model id must equal the GreenNode `path`.

**Deferred / parked** (intentional — see §5)
- **Data layer**: FDL MCP, Trino masked views, per-user masking, identity propagation. The source picker + Connect overlay are **UI-only**.
- **Auth**: no SSO — stub user "Linh".
- **Persistence**: library is localStorage; no server storage/retention policy.
- **Multi-tenant** (customers-as-config); **deployment** (local only).
- Upload **parsing** (metadata captured, content not yet parsed into context).

## 4. Run it

```bash
# 1. artifact export
cd services/artifacts && . .venv/bin/activate && uvicorn app.main:app --host 127.0.0.1 --port 8742
# 2. BFF (reads services/bff/.env → live GreenNode). Without a key it serves templates.
cd services/bff && npm run dev            # :8787
# 3. web
cd apps/web && npm run dev                # http://localhost:5173
```
Connect GreenNode: `docs/Connect-GreenNode.md` (+ `scripts/greennode-connect.sh`). OpenCode topology: `agent/README.md`.

## 6. Roadmap (next, in rough order)

1. **Data layer** — FDL MCP + Trino masked views + **per-user masking** + identity propagation (the parked "whole-risk" slice). Plugs in behind the existing source picker.
2. **Auth** — WSO2 SSO replacing the stub user; carry identity to the BFF.
3. **Server persistence + retention** — replace localStorage; artifact storage policy (D4).
4. **Refreshable reports** — live re-query under viewer identity (the recurring-token-burn flywheel).
5. **Deploy** — GreenNode (Compose → K8s); BFF-front the artifact service.
6. **Multi-tenant** — customers-as-config for the MaaS GTM.
7. Polish: VN-first shell, upload parsing, Studio inline edit, real sharing.
